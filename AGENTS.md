# Repository Guidelines

A quick reference for contributing to Blitz3D WASM Compiler.

## Project Structure

```
blitz3d-wasm/
├── Sources/
│   ├── Compiler/
│   │   ├── Lexer/          # Tokenization
│   │   ├── Parser/         # AST construction
│   │   ├── AST/            # Node definitions
│   │   ├── Preprocessor/   # Include handling
│   │   └── CodeGen/        # WASM generation
│   └── Runtime/
│       └── runtime.js      # Browser runtime
├── Tools/
│   └── wasm-cli/           # CLI compiler tool
├── Tests/
│   ├── CompilerTests/      # Swift XCTests
│   └── IntegrationTests/   # Node.js tests
└── docs/                   # Documentation
```

## Build & Test Commands

```bash
# Build the project
swift build

# Run all tests
swift test

# Run CLI with input file
swift run blitz3d-wasm input.bb -o output.wasm

# Compile with WAT output for debugging
swift run blitz3d-wasm input.bb -w -o output.wasm

# Run integration tests
cd Tests/IntegrationTests && node run_tests.js
```

## Coding Style

- **Swift**: Use Swift's default formatting (swift-format compatible)
- **JavaScript**: ES6+, 4-space indentation
- **WAT/WASM**: No manual editing; generated by compiler
- **Comments**: Minimal, functional code preferred
- **Naming**: camelCase for variables, PascalCase for types

## Testing Guidelines

- Swift tests: XCTest framework in `Tests/CompilerTests/`
- Integration tests: Node.js in `Tests/IntegrationTests/`
- Test files follow pattern: `*Tests.swift` or `run_tests.js`
- All PRs should include test coverage for new features
- Run `swift test` before submitting PR

## Commit & PR Guidelines

- **Commits**: Clear, descriptive messages. Reference issues with `#123`
- **PRs**: Include description of changes, testing performed
- **Branch naming**: `feature/description` or `fix/issue-number`
- **Reviews**: Require 1 approval before merge

## Key Development Notes

- The compiler processes: `Lexer → Parser → AST → CodeGen → WASM`
- Data statements inside functions require collection via `collectDataStatements()`
- Runtime uses Three.js for 3D graphics (loaded externally)
- Asset embedding supports `--embed-assets` flag
- Use `--manifest` to generate asset manifest JSON

## Configuration

- **Package Manager**: Swift Package Manager
- **Platform**: macOS (Swift), cross-compile to WASM
- **Dependencies**: None (self-contained)
- **WASM Memory**: 64KB initial, 128KB maximum
